Chosen Workflow:
The workflow I have chosen is building a RACI matrix. This is a document that tracks project tasks and how project stakeholders will interact with those tasks--determining responsibility, accountability, as well as who needs to be consulted and kept informed.

User: Project Manager

Input: The transcript of the internal project-kickoff meeting where project tasks and assignments are discussed.

Output: a RACI matrix, which is a chart with the list of project tasks, as well as associated stakeholders for those tasks. 

This will be helpful because it is a tedious part of every project, and one that needs to be completed for a project to be successful. However, because it's simple documentation it's often a workflow that is neglected. Having a RACI matrix generated, even if only a rough draft, would save a PM time on every project.
